# Contributing Guidelines

First off, thank you for considering contributing to this repository!

## Important Rules for Contributing

1.  **Do Not Modify Original Files:** You must never clean, rename, or delete existing original files (even if they appear to be duplicates like `app (1).py` or `rwc_gwl (1).py`).
2.  **Additions Only:** If you are implementing new features or fixes, please do so by adding new files or scripts, and do not change the existing source files directly.
3.  **Respect the Architecture:** Ensure that any new code belongs in its respective directory under `src/` (e.g., `src/hyper/` or `src/rwc_gwl/`).

## How to Submit a Contribution

1.  Fork the repository.
2.  Create a new branch for your feature or addition.
3.  Add your new files, ensuring no original files are altered.
4.  Run tests to ensure your additions do not break existing functionality.
5.  Submit a Pull Request with a clear description of the additions.
